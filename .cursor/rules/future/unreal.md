# Нереальные идеи (но вдруг)

Список амбициозных фич для конкуренции с Manus AI через нестандартные подходы.
Не roadmap — скорее "а что если".

---

## Локальность как киллер-фича

### Self-hosted marketplace
Пользователи делятся fine-tuned моделями для специфичных задач:
- `fastapi-expert.gguf` — обучен на FastAPI кодбейсах
- `react-typescript.gguf` — React + TS специализация
- `legacy-python2.gguf` — для миграции старого кода

### Федеративное обучение
Пользователи улучшают модели локально, делятся только градиентами (не кодом).
Модель становится лучше без утечки данных.

## Workflow-инновации

### "Спорящие агенты"
Coder генерирует код, Critic атакует, Coder защищается.
3 раунда дебатов → финальный код.
Медленнее, но качественнее для сложных задач.

### Агент-археолог
Анализирует git history:
- Кто писал этот код и когда
- Какие баги были связаны с этим файлом
- Почему выбрано именно это решение (из commit messages)

### Предсказание багов
ML-модель на истории проекта:
- "Этот файл меняется вместе с X в 80% случаев"
- "После изменений в auth.py обычно ломается tests/test_api.py"

## UX-эксперименты

### Голосовой режим
Whisper локально → задача → ответ голосом.
Кодинг без клавиатуры (для прототипирования).

### "Объясни как ребёнку"
Режим где агент объясняет каждый шаг простыми словами.
Для обучения джунов.

### Pair programming mode
Агент не пишет код, а комментирует твой код в реальном времени:
- "Тут потенциальный race condition"
- "Этот паттерн обычно приводит к N+1 queries"

## Монетизация без подписки

### Pay-per-task
Не $20/месяц, а $0.10 за сложную задачу.
Локально — бесплатно, облачные модели — платно.

### Bounty система
Пользователь платит за решение конкретной задачи.
Сообщество решает, лучшее решение получает bounty.

### Sponsorware
Новые фичи — сначала для спонсоров, через 3 месяца — всем.

## Безумные идеи

### Код как диалог
Вместо "сгенерируй функцию" — разговор:
```
User: Мне нужно кэшировать запросы
Agent: Redis или in-memory?
User: In-memory, но с TTL
Agent: Сколько записей максимум?
User: 10000
Agent: Вот LRU cache с TTL...
```

### Обратная связь от тестов
Тесты не просто pass/fail, а объясняют агенту что не так:
```
Test failed: expected User, got None
Context: database connection was not initialized
Suggestion: add setup_db() call before query
```

### Self-improvement loop
Агент анализирует свои ошибки:
- Какие задачи он решает плохо
- Какие промпты работают лучше
- Автоматически улучшает свои промпты
